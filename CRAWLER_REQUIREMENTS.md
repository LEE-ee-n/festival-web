# 크롤러 1차 검수 기준

## 목표

티켓 사이트 목록에서 축제 제목과 상세 URL만 수집하고, 중복을 제거해 로컬 JSON으로 출력한다.
운영 DB 저장과 상세 페이지 수집은 1차 범위에 포함하지 않는다.

## 필수 제한

- `.env`와 인증정보를 읽거나 수정하지 않는다.
- Supabase와 운영 DB에 직접 연결하지 않는다.
- 로그인, CAPTCHA, 접근 제한을 우회하지 않는다.
- `robots.txt`에서 금지한 경로를 요청하지 않는다.
- 요청 간격, 타임아웃, 최대 수집 개수를 설정한다.
- 결과는 `crawler-output/`에만 생성하고 Git에서 제외한다.
- 기존 `app/`, `components/`, `lib/`, `supabase/`를 수정하지 않는다.

## 출력 형식

```json
{
  "generated_at": "2026-07-18T00:00:00.000Z",
  "source_type": "ticket_site",
  "items": [
    {
      "title": "축제 제목",
      "source_url": "https://example.com/product/123",
      "source_type": "ticket_site",
      "discovered_at": "2026-07-18T00:00:00.000Z"
    }
  ]
}
```

## 필수 동작

- 사이트별 수집기는 독립된 어댑터로 분리한다.
- 상대 URL을 같은 사이트의 절대 URL로 변환한다.
- 빈 제목, 잘못된 URL, 다른 도메인 URL을 제외한다.
- 같은 URL을 중복 제거한다.
- 공백·대소문자·기호를 정리한 제목이 같으면 중복 후보로 판정한다.
- 결과를 항상 같은 기준으로 정렬한다.
- 한 페이지나 한 사이트의 실패가 전체 실행을 중단시키지 않는다.
- 실제 네트워크 없이 샘플 HTML로 테스트할 수 있어야 한다.

## 프로젝트 호환성

- 현재 프로젝트에 없는 `tsx`, `vitest`, `jest`를 임의로 요구하지 않는다.
- 기존 `node:test`와 TypeScript 실행 방식을 사용한다.
- `npm test`, `npm run lint`, `npm run typecheck`를 통과해야 한다.
- 기존 `.gitignore`는 덮어쓰지 않고 `crawler-output/` 항목만 추가한다.

## 필수 자동 테스트

- 정상 제목과 URL 수집
- 상대 URL 변환
- 빈 제목과 잘못된 URL 제외
- 동일 URL 중복 제거
- 정규화된 동일 제목 판정
- 일부 요청 실패 후 다음 요청 계속 처리
- 최대 수집 개수 준수
- 고정된 입력과 시각에서 동일한 출력 생성

## 1차 완료 조건

- 실제 사이트에 대량 요청하지 않는 샘플 어댑터가 동작한다.
- CLI에서 샘플 결과 JSON을 생성할 수 있다.
- 모든 필수 자동 테스트가 실제로 실행되어 통과한다.
- 생성·수정 파일과 미구현 범위가 문서에 기록되어 있다.

## 반자동 누락 탐색 목표

- 사용자가 일반 Chrome에서 NOL·Ticketlink·YES24 북마클릿을 실행한다.
- 전체 비교 CMD는 오늘 생성된 최신 JSON 세 개를 한 번에 처리한다.
- 파일이 빠지거나 오래된 파일이면 비교와 Discord 전송을 중단한다.
- 기존 `festivals`, `festival_candidates`, `festival_ticket_rounds`와 읽기 전용으로 비교한다.
- 동일 URL은 등록됨, 제목·별칭·날짜 유사는 확인 필요, 참조 없음은 누락 후보로 분류한다.
- 누락 후보와 확인 필요 명단은 CMD 실행마다 모두 보내며, 명시적으로 제외한 URL만 숨긴다.
- 자동 DB 저장, 상세 페이지 수집, Instagram 검색은 포함하지 않는다.
- `.env.local`을 사용하지 않고 Discord Bot의 기존 DPAPI Supabase 인증만 재사용한다.
- 자동 브라우저와 Windows 예약 실행을 사용하지 않는다.
- 후보 URL마다 실행 간 유지되는 `T-NNN` 번호를 부여한다.
- Discord에서 확인 필요는 `✅`, 누락 후보는 `🆕`로 표시한다.
- `!티켓제외 T-001, T-002` 명령은 정확한 URL만 제외하며 새 티켓 URL은 다시 보고한다.
