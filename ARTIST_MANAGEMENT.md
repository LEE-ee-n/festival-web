# 아티스트 관리

## 1차 완료 범위

- 신규 등록 전 기존 아티스트 유사 후보 검색
- 화면 표시 이름과 공식 영문명 입력
- 검색 결과가 없을 때 신규 아티스트 등록
- 전체 아티스트 ID·화면 표시 이름·`normalized_name`·별칭 조회
- ID·화면 표시 이름·`normalized_name` 오름차순·내림차순 정렬
- 목록과 검색 후보에서 편집 화면으로 즉시 이동
- ID를 제외한 화면 표시 이름·`normalized_name`·별칭 통합 수정
- 별칭 여러 개를 쉼표로 표시하고 입력
- 검색할 때 이름과 별칭의 공백 차이 무시

## 이름 규칙

- `artists.name`: 사이트에 표시하는 이름
- `artists.normalized_name`: 아티스트를 중복 없이 식별하는 값
- `artist_aliases.alias_name`: 한 아티스트의 다른 표기이며 별칭 하나당 한 행
- `normalized_name`은 영문 소문자와 숫자만 허용한다.
- `normalized_name`은 `NOT NULL`이며 전체 아티스트에서 유일해야 한다.
- 영문 이름은 화면에서 자동 정규화한다.
- 한글 이름만으로 `normalized_name`을 만들 수 없으면 공식 영문명을 입력해야 한다.

## 등록 경로

다음 경로는 동일한 아티스트 이름 정규화 규칙을 사용한다.

- 아티스트 관리 직접 등록
- 수집 후보 승인
- 축제 JSON 등록
- 축제 XLSX 등록

기존 아티스트가 확인되면 DB의 ID와 `normalized_name`을 사용한다. 신규 아티스트의 `normalized_name`이 없거나 형식이 잘못되면 등록을 중단한다.

## DB 마이그레이션

- `016_require_artist_normalized_name.sql`
  - 기존 누락·형식 오류·중복 검사
  - `NOT NULL`, 형식 검사, UNIQUE 인덱스 강제
- `017_update_artist_admin.sql`
  - 관리자 권한 확인
  - 표시 이름·`normalized_name`·별칭을 한 트랜잭션으로 수정
  - 다른 아티스트와 `normalized_name` 또는 별칭이 충돌하면 중단

## 화면 원칙

- 기존 아티스트의 표시 이름·`normalized_name`·별칭 수정은 아티스트 관리 페이지에서만 허용한다.
- 페스티벌 관리와 등록 단계에서는 기존 아티스트 마스터 정보를 읽기 전용으로 표시한다.
- 페스티벌 관리의 저장은 공연 날짜·시간·무대·상태 같은 축제별 연결 정보만 변경한다.
- 페스티벌 관리에서는 아티스트 검색·신규 등록·라인업 직접 추가를 허용하지 않는다.
- 특정 기존 페스티벌에 새 라인업을 연결할 때는 5단계 기존 페스티벌 수정 흐름에서 판별·검토·확정한다.
- 흰색 배경을 기본으로 사용한다.
- 영역은 중성 회색 테두리와 약한 그림자로 구분한다.
- 전체 목록의 기본 정렬은 ID 오름차순이다.
- ID와 수정 버튼은 줄바꿈하지 않는다.
- 별칭은 모두 표시하고 쉼표로 구분한다.

## 표 중심 관리 화면

- PC의 검색 후보는 `ID / 표시 이름 / normalized_name / 유사도 / 작업` 표로 표시한다.
- PC의 전체 목록은 `ID / 표시 이름 / normalized_name / 별칭 / 작업` 표로 표시한다.
- 전체 목록에서 `수정`을 누르면 별도 수정 카드로 이동하지 않고 선택한 행의 이름·`normalized_name`·별칭이 입력칸으로 전환된다.
- 한 번에 한 아티스트만 편집하며 편집 행에서 `저장`·`취소`·`삭제`를 처리한다.
- 후보의 `목록에서 수정` 또는 신규 등록 성공 후에는 전체 목록의 해당 행을 선택하고 화면을 이동한다.
- 모바일에서는 후보와 전체 목록을 카드로 표시하고 선택 카드에서 같은 편집 상태와 이벤트를 사용한다.
- 오류·성공 문구는 페이지 공통 위치에 한 번만 표시한다.

## 검증

- 운영 DB의 비어 있거나 잘못된 `normalized_name`: 0건
- 운영 DB의 중복 `normalized_name`: 0건
- 린트, 타입 검사, 자동 테스트 통과

## 이후 개선

- 실제 PC·모바일에서 표·카드 배치와 선택 행 이동을 확인한다.
- 실제 운영 중 발견되는 검색 표현과 편집 편의 기능은 이 페이지에 추가한다.
- 아티스트 수가 크게 늘면 서버 페이지네이션과 가상 스크롤을 별도 검토한다.
