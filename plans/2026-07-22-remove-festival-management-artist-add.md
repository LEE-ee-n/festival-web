# 페스티벌 관리 라인업 아티스트 추가 제거 계획

- 작성일: 2026-07-22
- 상태: 코드 구현 완료·실제 화면 확인 대기
- 승인일: 2026-07-22
- 대상: `/admin/festivals/[id]/lineup`
- DB 변경: 없음

## 한눈에 보는 요약

- 페스티벌 관리의 `라인업 아티스트 추가` 영역을 완전히 제거한다.
- 이 화면에서는 기존 라인업의 공연 일정 수정과 라인업 삭제만 허용한다.
- 아티스트 생성·공통정보 수정은 아티스트 관리 페이지에서만 한다.
- 기존 페스티벌에 새 라인업을 연결하는 현재 안전 경로는 5단계 기존 페스티벌 수정 작업이다.

중요: 현재 아티스트 관리 페이지는 아티스트 마스터 생성·수정 기능만 있고 특정 페스티벌 연결 기능은 없다. 이번 제거 후 직접 관리 화면에서 즉시 라인업을 추가할 수 없으며, 5단계 기존 페스티벌 수정 흐름을 사용해야 한다. 아티스트 관리 페이지에서 페스티벌까지 연결하는 기능은 별도 계획 대상이다.

## 확정 범위

1. 라인업 탭의 `ArtistAddSection` 렌더링과 import를 제거한다.
2. 기존 아티스트 검색·선택·추가 상태와 이벤트를 `useFestivalArtists`에서 제거한다.
3. 신규 아티스트 이름·`normalized_name`·별칭 입력과 임시 행 생성 로직을 제거한다.
4. 사용되지 않는 `ArtistAddSection.tsx` 파일을 삭제한다.
5. 기존 라인업 표의 읽기 전용 아티스트 정보, 일정 수정, 일정 저장, 삭제는 유지한다.
6. 발표·정정 기록과 전체 미저장 일정 저장은 유지한다.

## 제거되는 상태·동작

- 아티스트 검색어와 검색 결과
- 기존 아티스트 선택 후 임시 라인업 행 추가
- 신규 아티스트 입력 모드
- 신규 아티스트 임시 행 생성
- 라인업 추가용 날짜·시간·무대 입력 상태
- `addSectionProps`

## 유지할 규칙

- 아티스트 공통정보는 아티스트 관리 페이지에서만 생성·수정한다.
- 페스티벌 관리에서는 아티스트 마스터 정보를 읽기 전용으로 표시한다.
- 공연 일정은 페스티벌별 정보이므로 이 화면에서 수정할 수 있다.
- 기존 라인업 관계 삭제는 발표·정정 감사 기록을 통해 저장한다.
- 새 라인업 연결은 기존 페스티벌 5단계 수정 흐름에서 검토·확정한다.
