# [완료] 페스티벌 관리 티켓 변경 메타데이터 입력 제거 계획

- 작성일: 2026-07-22
- 상태: 완료
- 승인일: 2026-07-22
- 대상: `/admin/festivals/[id]/lineup` 티켓 관리 탭
- DB 변경: 없음

## 한눈에 보는 요약

- 티켓 관리 상단의 `변경 출처 URL (선택)`, `변경 메모 (선택)` 입력칸을 삭제한다.
- 티켓 추가·수정·삭제 기능과 입력 항목은 그대로 유지한다.
- 제거 후 직접 티켓 작업의 감사 로그에는 별도 출처 URL과 메모를 전달하지 않는다.

## 목적

티켓 관리에서 실제 작업에 사용하지 않는 공통 변경 메타데이터 입력칸을 제거해 화면을 단순화한다.

## 확정 범위

1. `TicketTab`에서 두 입력칸과 관련 props를 제거한다.
2. `useFestivalTickets`에서 `ticketSourceUrl`, `ticketNote` 상태와 setter를 제거한다.
3. 티켓 추가·수정·삭제 함수 호출에서 선택 메타데이터 인자를 제거한다.
4. 기존 티켓 회차·표시 이름·오픈 시간·예매처·가격·예매 링크 관리 기능은 유지한다.

## 예상 수정 파일

- `app/admin/festivals/[id]/lineup/components/TicketTab.tsx`
- `app/admin/festivals/[id]/lineup/hooks/useFestivalTickets.ts`
- 이 계획 문서와 `PROJECT_STATUS.md`

## 유지할 규칙

- 티켓 추가·수정·삭제는 기존 감사 RPC를 계속 사용한다.
- 감사 이벤트 자체는 생성하며 `source_url`, `note`만 비어 있게 둔다.
- 신규·기존 페스티벌 5단계 등록 흐름의 출처 URL 보존 규칙은 변경하지 않는다.
- 썸네일 변경 출처 URL·메모 입력은 이번 작업 범위가 아니므로 유지한다.
