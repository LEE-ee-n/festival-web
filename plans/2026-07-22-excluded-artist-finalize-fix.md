- DB 변경: 없음

## 한눈에 보는 요약

- `Artist name is empty.`의 원인은 삭제·제외한 빈 아티스트도 변경 미리보기에 남아 최종 RPC로 전달된 것이다.
- `match_status = excluded`인 항목을 공통 변경 미리보기에서 완전히 제외한다.
- 제외 항목이 최종 아티스트 배열에 들어가지 않는 회귀 테스트를 추가한다.

## 구현 범위

1. `createFestivalUpdatePreview()`가 `excluded` 아티스트를 라인업 변경으로 만들지 않게 한다.
2. 최종 반영은 기존처럼 미리보기에서 선택된 항목만 사용하므로 제외 항목이 RPC로 전달되지 않게 한다.
3. 기존 라인업을 자동 삭제하지 않는 규칙은 유지한다.
4. 빈 신규 아티스트를 임의 보정하거나 자동 등록하지 않는다.

## 검증

- 제외된 빈 아티스트가 미리보기 라인업 항목에 나타나지 않음
- 기존·신규 활성 아티스트 미리보기는 유지
- 관련 단위 테스트, typecheck, 변경 파일 ESLint
- 현재 임시작업에서 최종 확정 재시도

## 완료 조건

- 제외 아티스트가 최종 RPC 아티스트 배열에 포함되지 않는다.
- `Artist name is empty.` 오류 없이 현재 작업을 최종 반영할 수 있다.

## 제외 범위

- DB·RPC 변경
- 이름이 실제로 비어 있는 활성 신규 아티스트의 자동 보정
- 다른 최종 반영 오류의 추측 수정

## 후속 개선점·참고사항

- 최종 반영 재시도에서 다른 DB 검증 오류가 나오면 실제 문구를 기준으로 별도 판단한다.
