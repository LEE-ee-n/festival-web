# 기존 페스티벌 4단계 필수정보 검증 계획

- 작성일: 2026-07-22

- 승인일: 2026-07-22
- 대상: 기존 페스티벌 수정 4단계 → 5단계 이동
- DB 변경: 없음

## 한눈에 보는 요약

- 선택한 라인업 변경이 있으면 4단계에서 발표 기록 필수정보를 검사한다.
- 발표일·출처 등 필수정보가 비어 있으면 오류를 바로 표시하고 5단계로 넘어가지 못하게 한다.
- 최종 확정의 기존 검증도 이중 안전장치로 유지한다.

## 확인한 원인

- 4단계에는 `공식 발표일`, `출처 URL` 입력란이 있다.
- 공통 `validateLineupWork()`는 현재 최종 확정 함수에서만 호출된다.
- 단계 이동 함수에는 해당 검증이 없어 빈 필수값으로도 5단계 진입이 가능하다.

## 구현 범위

1. 4단계에서 `다음`을 누를 때 선택된 라인업 변경이 있는지 확인한다.
2. 변경이 있으면 기존 `validateLineupWork()`로 발표·정정 필수정보를 검증한다.
3. 오류가 있으면 현재 4단계를 유지하고 화면 오류 영역에 정확한 문구를 표시한다.
4. 검증 통과 후에만 임시저장하고 5단계로 이동한다.
5. 이전 단계 이동은 차단하지 않는다.
6. 최종 확정의 기존 검증은 삭제하지 않는다.

## 예상 수정 파일

- `app/admin/festivals/import-json/StagedFestivalUpdate.tsx`
- `FESTIVAL_INGESTION_FLOW.md`
- 이 계획 문서와 `PROJECT_STATUS.md`

## 검증

- 발표 + 발표일 없음: 5단계 이동 차단
- 발표 + 출처 URL 없음: 5단계 이동 차단
- 발표 필수정보 모두 입력: 5단계 이동 허용
- 정정 + 출처와 사유 모두 없음: 이동 차단
- 선택된 라인업 변경 없음: 기존처럼 이동 허용
- 이전 버튼: 기존처럼 이동 허용
- `tests/lineupWork.test.ts`, typecheck, 변경 파일 ESLint
