# 기존 페스티벌 최종 검토 한 줄 배치 계획

- 작성일: 2026-07-22
- 상태: 코드 구현 완료·화면 확인 대기
- 승인일: 2026-07-22
- 대상: 기존 페스티벌 수정 5단계 `최종 반영 검토`
- DB·저장 로직 변경: 없음

## 한눈에 보는 요약

- 초록·연두색을 모두 제거하고 흰색·회색만 사용한다.
- `기본정보 / 아티스트·타임테이블 / 티켓 / 최종 확정 버튼`을 데스크톱 한 줄에 배치한다.
- 최종 확정 버튼은 연한 회색으로 바꾸되 기존 실행 조건과 기능은 유지한다.

## 확정 범위

1. 최종 검토 전체 영역을 흰색 배경과 회색 테두리로 변경한다.
2. 제목과 안내문을 검정·회색 조합으로 변경한다.
3. 세 요약 항목과 최종 확정 버튼을 데스크톱 4열 한 줄로 배치한다.
4. 버튼은 요약 항목과 같은 높이에 맞추고 연한 회색 배경을 사용한다.
5. 좁은 화면에서는 내용이 잘리지 않도록 1열 또는 2열로 자연스럽게 줄바꿈한다.
6. `finalize()`, 비활성 조건, 반영 중 문구, 선택 건수 계산은 변경하지 않는다.

## 수정 파일

- `app/admin/festivals/import-json/StagedFestivalUpdate.tsx`
- 이 계획 문서
- 완료 후 `PROJECT_STATUS.md`

- typecheck와 해당 파일 ESLint
- 선택 0건일 때 버튼 비활성 확인
- 선택 항목이 있을 때 기존 최종 확정 동작 유지 확인
- 데스크톱 한 줄 배치와 모바일 줄바꿈 확인

## 완료 조건

- 초록색 계열이 남지 않는다.
- 데스크톱에서 세 요약 항목과 버튼이 한 줄에 표시된다.
- 저장 동작과 검증 조건에 변화가 없다.

## 제외 범위
